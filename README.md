# Website Login & Email Capture (Business Analysis)

## Project Overview

This project demonstrates the **business analysis and requirements documentation process** for a simple website login page designed to collect and manage customer email addresses.

The project showcases the planning and documentation activities typically performed by **Business Analysts** and **Project Coordinators** before software development begins. It includes business requirements, user stories, use cases, process flows, and wireframes that communicate stakeholder needs to the development team.

---

## Business Problem

Organizations often require users to create an account or submit an email address to access online services, receive communications, or manage customer relationships.

The success of these applications depends on clearly documented business requirements that align stakeholder expectations with the software development process.

This project demonstrates how a Business Analyst documents functional requirements for a simple login and email capture feature.

---

## Project Objectives

### 1. Define Business Requirements

Document the business need, project objectives, scope, assumptions, and functional requirements.

**Deliverables**

- Business Requirements Document (BRD)
- Functional requirements
- Business rules
- Project assumptions
- Scope definition

---

### 2. Document User Requirements

Capture user needs using Agile-style user stories.

**Deliverables**

- User Stories
- Acceptance Criteria
- Business Value Statements

---

### 3. Model User Interactions

Develop use cases describing how users interact with the application.

**Deliverables**

- Use Case Document
- Primary and alternate flows
- Preconditions
- Postconditions
- Exception scenarios

---

### 4. Document Business Processes

Illustrate the user workflow from accessing the login page through successful email capture.

**Deliverables**

- Process Flow Diagram
- Decision points
- Process steps

---

### 5. Create Wireframes

Develop low-fidelity interface mockups to communicate the intended user experience.

**Deliverables**

- Login Page Wireframe
- Email Capture Screen
- Password Reset

## Business Questions Addressed

This project demonstrates how Business Analysts translate business needs into software requirements by answering questions such as:

- What problem is the application solving?
- Who will use the system?
- What information should be collected?
- What happens when users submit invalid information?
- How should the system respond after successful login?
- What requirements must developers implement?
